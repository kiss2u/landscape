# Contributing to Landscape

Thanks for your interest in contributing. Landscape is an eBPF-based Linux Router Platform — contributions across Rust, C/eBPF, TypeScript/Vue, and documentation are all welcome.

[简体中文](./CONTRIBUTING.zh.md)

## Code of Conduct

Be respectful. Keep discussions technical and constructive.

## How to Contribute

### Reporting Bugs

Use the [Bug Report template](https://github.com/ThisSeanZhang/landscape/issues/new?template=bug-报告模板---bug-report-template.md). Include:

- OS, kernel version, software version
- Exact steps to reproduce
- Expected vs actual behavior
- Relevant logs or screenshots

### Requesting Features

Use the [Feature Request template](https://github.com/ThisSeanZhang/landscape/issues/new?template=功能请求模板---feature-request-template.md). Describe the use case and why the current behavior is insufficient.

### Suggesting Enhancements

Use the [Feature Request template](https://github.com/ThisSeanZhang/landscape/issues/new?template=功能请求模板---feature-request-template.md). Provide clear motivation and impact.

### First-Time Contributors

Look for issues labeled `good first issue` or `help wanted`.

## Development Setup

See [BUILD.md](./BUILD.md) for the full local development guide. Quick start:

```bash
# Install system dependencies
sudo apt-get install -y cmake clang curl gcc llvm make pkg-config libelf-dev libclang-dev zlib1g-dev zstd

# Install pnpm and frontend dependencies
pnpm install --frozen-lockfile

# Generate TypeScript API bindings (required before frontend work)
./gen_ts_bindings.sh
```

## Before Submitting a PR

**Formatting is required before every pull request.** Run the formatting script to ensure your code passes CI:

```bash
./fmt.sh
```

This runs all three formatters (Rust, C/eBPF, Frontend). You can also format only a specific language:

```bash
./fmt.sh --rust      # cargo fmt
./fmt.sh --c         # clang-format-18
./fmt.sh --frontend  # prettier
```

Then run the full PR checklist:

```bash
cargo test --workspace
pnpm --filter landscape-webui exec prettier --check "src/**/*.{vue,ts,js,json,css,scss}"
pnpm --filter landscape-webui build
```

### Regenerating API Bindings

If you changed backend OpenAPI routes or schemas, regenerate the TypeScript API client:

```bash
./gen_ts_bindings.sh
```

This exports `openapi.json` locally.

## Code Conventions

### General

- Follow the existing style in the file you are editing.
- Do not introduce new dependencies without strong justification.
- Keep PRs focused: one logical change per PR.

### Rust

- Format with `./fmt.sh --rust` or `cargo fmt`. The project uses a [`.rustfmt.toml`](./.rustfmt.toml).
- Write tests for new logic. Run `cargo test --workspace`.
- Derive `Serialize`/`Deserialize` via `landscape-common` conventions when adding API types.
- If you change OpenAPI routes or schemas, run `./gen_ts_bindings.sh` to update local bindings.

### C / eBPF

- Format with `./fmt.sh --c` or `clang-format-18`. The project uses [`.clang-format`](./.clang-format).
- eBPF programs live under `landscape-ebpf/src/bpf/`. Do not format the generated `vmlinux.h`.
- Keep BPF programs compatible with the kernel versions documented in [BUILD.md](./BUILD.md).

### TypeScript / Vue

- Format with `./fmt.sh --frontend` or `prettier`.
- Use TypeScript strictly. The frontend type-checks with `vue-tsc`.
- API client code in `landscape-types/` is auto-generated by [orval](https://orval.dev/). Do not edit it directly.

### Commit Messages

No strict format, but prefer:

```
<area>: <brief description>

<optional body>
```

Examples: `ebpf: fix NAT4 rule leak on interface down`, `webui: add dark mode toggle`.

## Project Structure

| Directory | Purpose |
|---|---|
| `landscape/` | Core library (config, NAT, DNS, eBPF integration) |
| `landscape-webserver/` | HTTP/HTTPS server, REST API, WebSocket |
| `landscape-ebpf/` | eBPF C programs + Rust userspace loader |
| `landscape-database/` | SeaORM schema + DuckDB metrics |
| `landscape-gateway/` | Pingora-based gateway |
| `landscape-dns/` | hickory-based DNS server |
| `landscape-common/` | Shared utilities, OpenAPI support |
| `landscape-macro/` | Procedural macros |
| `landscape-protobuf/` | Protobuf definitions |
| `landscape-types/` | Auto-generated TypeScript API client |
| `landscape-webui/` | Vue 3 + Vite frontend |
